FIX48 — почему у тебя 400/404 и что делать

По твоим DevTools видно, что сервер не обновлён:
- GET /api/profile/templates -> 404  (этой ручки нет в старом бэке)
- POST /api/robux/inspect и /api/robux/order_reserve -> 400 с текстом «не удалось распознать ссылку…»
  (старый бэк понимает только link/id и не умеет username-режим)

Это означает: обновился ФРОНТ (static/app.js, styles.css), а Python-бэк (app.py) на сервере остался старым или его не перезапустили.

ПРОВЕРКА (самое простое): открой в браузере
  /api/version
Должен вернуться JSON вида:
  { ok: true, build: "fix48", version: "fix48-..." }
Если build не fix48 — ты всё ещё на старом бэке.

ЧТО ДЕЛАТЬ:
1) Замени на сервере ВЕСЬ проект из этого архива, а не только папку static.
   Важно, чтобы обновился файл app.py.
2) Перезапусти процесс сервера.
   Варианты:
   - если запускаешь руками: останови текущий процесс и запусти заново
       uvicorn app:app --host 0.0.0.0 --port 8000
   - если у тебя systemd:
       sudo systemctl restart <имя_сервиса>
   - если Gunicorn:
       sudo systemctl restart gunicorn
   - если Render/Heroku/Vercel-подобные: сделай Deploy заново (Manual Deploy / redeploy), чтобы подтянулся новый app.py.

После перезапуска:
- /api/profile/templates перестанет быть 404 и шаблоны начнут сохраняться.
- robux по нику перестанет падать с «распознать ссылку», потому что новый бэк умеет username-режим.

ПРИМЕЧАНИЕ:
Ошибки /api/topup/config 500 и banner 404 не влияют на robux. Это отдельные части сайта (конфиг пополнений/баннер).
