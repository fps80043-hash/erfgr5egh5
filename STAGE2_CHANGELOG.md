## 1. Maintenance fix — админ больше не вылетает

**Было:** при включении тех. работ middleware проверял только наличие токена сессии. Любой авторизованный юзер бэкенду показывался как «свой», но фронтенд маршрутизировал на 503-страницу. Админ выпадал на «Тех. работы» вместе со всеми, даже на `/v2`. Аккаунт «вылетал» потому что некоторые API возвращали 503 → фронт думал, что юзер деавторизован.

**Стало:** middleware теперь:
- Делает запрос в БД `SELECT is_admin` для текущего uid
- Если `is_admin=1` — даёт **полный байпас везде** (любые API, любые страницы)
- Для `/`, `/v2`, `/v2/...` — пропускает запрос дальше, чтобы фронтенд показал свой красивый оверлей вместо голой HTML-страницы 503
- Для обычных юзеров пускает только `/api/auth/*`, `/api/balance`, `/api/notifications/unread_count` — это значит сессия не инвалидируется и юзер не «вылетает»

**Фронт:** новый `<div class="maint-overlay">` с пульсирующим эмодзи 🔧 поверх всего сайта. Опрашивает `/api/site/status` каждые 60 секунд. Админу оверлей не показывается (видит обычный сайт + ремарку, что он админ).

## 2. Лента популярных категорий (как на 3-м скриншоте)

3D-карусель в стиле Cover Flow:
- Центральная карточка крупная, по бокам — уменьшенные с наклоном (`rotateY 18deg`) и затемнением
- Дальние карточки — ещё мельче и в blur
- Стрелки навигации (◀ ▶) по бокам, точки внизу
- Авто-ротация каждые 5 сек (пауза при наведении мыши)
- Поддержка клавиатуры (← →)
- Клик на боковую карточку — переключение влево/вправо, клик по центральной — переход в категорию
- Адаптивно для мобилок (2 видимые карточки вместо 5)

Берёт первые 8 категорий с баннерами из `SHOP.categories`, сортированных по `sort`. Показывается только если категорий ≥ 3.

## 3. Размер карточек товаров — синхронизирован с v1

`.game-art` был `aspect-ratio:1` (квадрат) — теперь **`16/9`**, как в v1 `.product-banner`. Картинки больше не «облезают», красиво обрезаются как в классике.

## 4. Авто-движение баннеров

Баннеры (`.banner-row`) теперь:
- Каждые 4.5 сек автоматически прокручиваются на одну карточку
- Зацикливаются (на конце скроллит к началу)
- Пауза при наведении мыши на ленту
- Пауза при ручном скролле/touch (на 2.5 сек после последнего касания)
- Каждый баннер мягко «парит» (анимация `bnFloat` с задержками)
- При клике пытается перейти в соответствующую категорию (Robux → Roblox, etc.)

## 5. Анимация всего

- **Карточки игр**: появляются по мере прокрутки (Intersection Observer), плавно поднимаются на `translateY(16px)` → 0 с opacity 0 → 1
- **Свечение карточек**: при ховере появляется мягкое фиолетовое сияние через `box-shadow`
- **Section headers**: подчёркивание-градиент под заголовками
- **Маленькая кнопка ↑** в правом нижнем углу при скролле > 400px

## 8. Безопасность в профиле — теперь работает

Раньше была заглушка с редиректом на классику. Теперь все 3 секции рабочие:

**🔑 Смена пароля** — `/api/security/password`. Текущий + новый + повтор, валидация, поясняющие сообщения.

**✉️ Смена email** — двухшаговый процесс:
1. Вводишь новый email + текущий пароль → `/api/security/email_start` (отправляет код на новый email)
2. Вводишь код → `/api/security/email_confirm` (применяет смену)

**🛡 2FA** — двухшаговый процесс:
- Если 2FA выключена: пароль → код → `/api/user/twofa_enable_start`, `/api/user/twofa_enable_confirm`
- Если включена: пароль → `/api/user/twofa_disable` с подтверждением

Все ошибки показываются inline под формой, успех — toast-уведомлением.
