# Claude — changelog

Этот архив содержит проект rbx-site с правками, сделанными в чате.
Перед накатом на прод — обязательно прочитай раздел «Что сделать ДО деплоя»
в самом конце.

## Что изменилось

### Безопасность платежей (P0)

- **Удалена утечка исходников бэкенда** из публичной директории
  `/static/`. Файлы `static/main.py`, `static/app.py` и старые версионные
  `static/app.5.3.*.{js,css}`, `static/*.fix41.*`, вложенная папка
  `static/static/` — все убраны.
- **CryptoPay-вебхук** (`main.py`, функция `_cryptopay_webhook_impl`):
  проверка HMAC-подписи теперь обязательна всегда. Раньше при отсутствии
  заголовка `crypto-pay-api-signature` была ветка «no signature → accept»,
  через которую можно было зачислить произвольный pending-топап. Также
  добавлена сверка `asset` (валюты) из payload с `topups.fiat_currency`
  и проверка положительной суммы.
- **Гонка начисления баланса** (`main.py`, `_credit_topup_once`):
  read-modify-write `SELECT balance` + `UPDATE balance=?` заменён на
  атомарный `UPDATE users SET balance = balance + ? WHERE id=?`
  в обеих ветках (Postgres и SQLite). При параллельных вебхуках
  начисления больше не теряются.

### YooKassa полностью удалена

- Из `main.py` вырезаны: ветка `if method == "yookassa"` в создании
  платежа, обработчик `/api/yookassa/webhook`, env-переменные
  `YOOKASSA_SHOP_ID` / `YOOKASSA_SECRET_KEY`, функция
  `yookassa_enabled()`, запись в конфиге методов. Минус ~7 КБ кода.
- Несколько остаточных строк в `static/app.js` (логика маршрутизации
  методов оплаты) оставлены сознательно — они безвредны, метод
  `yookassa` теперь просто никогда не выберется. Уберу в следующей
  итерации вместе с переделкой блока пополнения.

### Мёртвый код удалён

- `app.py` (10 326 строк) — был дублем `main.py`. Боевой entrypoint
  всегда был `main.py` (см. `run.py` и `Procfile`), а `app.py` мог
  ввести в заблуждение при правках.

### Фронтенд (`static/app.js`)

- **Часовой пояс:** UI-селектор удалён из настроек. Под капотом
  `_getTz()` теперь всегда возвращает текущий пояс устройства через
  `Intl.DateTimeFormat().resolvedOptions().timeZone`. Старый ключ
  `rst_tz` из localStorage чистится одноразово.
- **Переключатель в настройках починен.** Был баг: внутри `<label>`
  лежал `<input style="display:none">` и тег `<style>` (невалидный
  HTML, часть браузеров игнорировала). Стили вынесены в `<head>`,
  input сделан кликабельным через `opacity:0`. Теперь работают оба
  тоггла — и звук, и «Предотвратить закрытие окон».
- **Звук теперь реально выключается.** Добавлена функция
  `_soundEnabled()` и гейт в `_tone()` и `_playCaseTick()` —
  две точки входа всего звукового движка.

### Новый интерфейс v2

- **Роут** `GET /v2`, `/v2/`, `/v2/{subpath}` в `main.py` — отдаёт
  `templates/v2.html`. Любая ссылка вроде `/v2/catalog` ведёт на
  тот же v2 (внутренний роутинг будем делать на JS позже).
- **Шаблон** `templates/v2.html` — самостоятельная страница, ноль
  зависимостей от v1. iOS-вайб: squircle-карточки 22–28px, frosted
  header через `backdrop-filter`, типографические обложки в
  фирменных цветах брендов (без чужих лого), 7 игр в каталоге
  (Roblox / Telegram / Discord / Steam / Brawl Stars / Clash Royale /
  Clash of Clans), баннер-карусель с горизонтальным snap-скроллом,
  плавные анимации появления карточек.
- **На v1** в правом нижнем углу плавающая кнопка «✨ Попробовать v2»
  (одна вставка перед `</body>` в `templates/index.html`).
- **На v2** в левом верхнем углу пилюля «← Классика».


## Что сделать ДО деплоя

1. **Создай git-ветку** перед накатом: `git checkout -b claude-fixes`.
   Коммить по одному изменению за раз, чтобы был быстрый откат.
2. **Локальный запуск:** `python run.py` → открой `http://localhost:8000`,
   проверь что старая версия открывается без ошибок и что появилась
   кнопка «✨ Попробовать v2». Кликни — должна открыться `/v2`.
3. **Поменяй секреты в Railway env:** `CRYPTO_PAY_API_TOKEN`,
   `CRYPTO_PAY_WEBHOOK_TOKEN`, любые другие платёжные/админские ключи.
   Раз `main.py` лежал в `/static/` публично — считаем, что код мог
   быть скачан, ротация обязательна.
4. **Удали из Railway env** переменные `YOOKASSA_SHOP_ID` и
   `YOOKASSA_SECRET_KEY` — они больше нигде не читаются.
5. **Проверь логи Railway** на запросы `GET /static/main.py` или
   `GET /static/app.py` за период, что они там лежали. Если хоть
   один не-твой IP стучался — это подтверждённая компрометация:
   проведи аудит балансов на аномальные топапы.
6. **Тестовый платёж в CryptoPay на маленькую сумму** —
   убедись, что после правок начисление баланса работает на реальной
   оплате. Я фиксил вебхук вслепую, без возможности прогнать его на
   твоей реальной CryptoPay-конфигурации.


## Что НЕ протестировано

Чтобы не было сюрпризов:

- Я не запускал проект целиком. Я проверил только синтаксис Python
  (`py_compile`), JS (`node --check`) и Jinja-парсинг шаблонов.
- Колонки `fiat_cents` / `fiat_currency` в таблице `topups` я видел
  в `INSERT` на ~262 строке `main.py`, но в твоей **проде** схема
  может отличаться. Если этих колонок нет, сверка суммы в
  CryptoPay-вебхуке вернёт `expected_rub = None` и **пропустит**
  проверку (это безопасное поведение, лучше пропустить чем сломать
  платежи). Перепроверь на проде:
  `SELECT column_name FROM information_schema.columns WHERE table_name='topups';`
- `static/app.js` тестировался только синтаксически, не кликался
  в браузере.
- `templates/v2.html` я не открывал в браузере вживую.

Поэтому локальный запуск перед деплоем — обязателен.
