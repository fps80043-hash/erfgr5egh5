# Переход на PostgreSQL (Railway)

## Зачем?
SQLite блокирует базу данных при параллельных записях → ошибка "database is locked".
PostgreSQL поддерживает сотни одновременных подключений без блокировок.

## Шаги на Railway:

### 1. Добавить PostgreSQL
- Зайди в проект на Railway
- **+ New** → **Database** → **PostgreSQL**
- Railway автоматически создаст базу данных

### 2. Подключить к сервису
- Перейди в свой web-сервис (где main.py)
- **Variables** → **New Variable**
- Имя: `DATABASE_URL`
- Значение: скопируй из PostgreSQL сервиса → **Connect** → **Connection URL**  
  (формат: `postgresql://user:pass@host:5432/railway`)

### 3. Деплой
- После добавления переменной Railway автоматически передеплоит
- Код автоматически определяет `DATABASE_URL` и использует PostgreSQL
- Все таблицы создаются автоматически при первом запуске

### 4. Проверка
В логах Railway должно появиться сообщение без ошибок при старте.
SQLite файл `/data/data.db` больше не используется.

## Что меняется?
- ✅ Нет "database is locked" ошибок
- ✅ 100+ одновременных пользователей
- ✅ Данные хранятся в надёжной БД Railway
- ✅ Автоматический бэкап (Railway PostgreSQL)

## Обратная совместимость
Если `DATABASE_URL` не установлен — сайт продолжит работать на SQLite.
