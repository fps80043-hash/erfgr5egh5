# v2.html — аддитивные доделки (паритет с v1)

**Дата:** 2026-04-26  
**Что было:** оригинальный v2 (1571 строк) — маркетплейс с баннерами, сеткой игр и subcategory-режимом для Robux. Дизайн сохранён **полностью**.  
**Что стало:** 2841 строк — добавлено ~1270 строк нового CSS+HTML+JS, удалено всего 25 строк (заменены на расширенные версии).

## Что было удалено (и почему)

Все 25 строк, которые я удалил, **были заменены на расширенные версии** того же элемента — никакой функционал не пропал:

1. **Subnav** «Раздача / Отзывы / Поддержка / Блог» (`<a href="#">`) → теперь рабочие onclick-ссылки на модалки.
2. **User-menu** «Профиль (классика) / Мои покупки (классика) / Настройки (классика)» — все вели на `/`. Заменены на: Профиль (открывает модалку), Пополнить, Кейсы, AI инструменты, Поддержка, Отзывы.
3. **submitAuth** — старая версия не работала с капчей и не поддерживала 2FA или email-подтверждение регистрации. Перепиcана.

Дизайн (баннеры, цвета, шрифты, скруглления, .game карточки, .pmodal, .tmodal, admin overlay, .rbx-calc и т.д.) — **не тронут**.

## Единый вход v1 ↔ v2

Уже работал в оригинале: бэкенд ставит `SESSION_COOKIE` с `path="/"`, а все fetch-запросы используют `credentials: 'same-origin'`. Юзер залогиненый на v1 автоматически видит свою сессию на v2 и наоборот. Никаких изменений в бэке не нужно.

## Новая капча — drag-to-confirm

Раньше: математический ввод «5+3=?». Теперь — слайдер. Пользователь тянет слайдер до конца (95% позиции), JS вычисляет `a+b` локально из ответа `/api/captcha/challenge` и при сабмите отдаёт его в поля `captcha_token` + `captcha_answer`. Бэковый `_captcha_verify` принимает их без изменений.

Поддерживается мышь + тач (`touchstart/move/end`). При недотягивании — слайдер откатывается. После верификации меняется цвет и текст («✓ Подтверждено»).

## Что добавлено (по разделам)

### Header
- Колокольчик уведомлений (только для залогиненных) с пульсирующим счётчиком непрочитанных. Поллинг каждые 30 сек.
- Панель уведомлений (350px, top-right): список последних 30, отметка прочитанного по клику, «Прочитать все», авто-закрытие при клике вне.

### Subnav
Все ссылки рабочие: Главная, Кейсы, Отзывы, Поддержка, AI.

### User-menu
Все пункты открывают модалки (не редиректят на /):
- 👤 Профиль
- 💳 Пополнить
- 🎲 Кейсы
- 🤖 AI инструменты
- 💬 Поддержка
- ⭐ Отзывы
- 🛠 Управление магазином (если admin)

### Auth modal (расширен)
- 3 шага: основной (вход/регистрация со слайдер-капчей), 2FA (код из письма), confirm-email (после регистрации).
- Корректная обработка `needs_2fa` от бэка.
- После регистрации остаёмся на v2 — не редиректим на v1.

### Profile modal (новый)
- Большой аватар с кликом для смены, мини-крестик для сброса (`/api/user/avatar` POST multipart, `/api/user/avatar/reset`).
- Бейджи Premium / Admin.
- Балансы и Premium до даты.
- 3 KPI: Пополнено / Потрачено / Покупок (из `/api/balance/history` и `/api/purchases`).
- Табы: Обзор / Покупки / Безопасность.

### Cases / Inventory modal (новый)
- Список кейсов из `/api/cases/config` (free / paid / money).
- Полноценная рулетка: 48 элементов с CSS transition 5.2 секунды, tier-стилизация (1–4), пульсирующие указатели сверху и снизу.
- После открытия — экран победы с emoji приза.
- Вкладка Инвентарь: список призов с кнопками Use / Delete.

### Reviews modal (новый)
- Полный список через `/api/reviews`.
- Если юзер не оставлял отзыв (`/api/reviews/my`) — форма с 5-звёздной оценкой и текстом.
- Сабмит через `/api/reviews/create`.
- Отзывы с аватарами и бейджами Premium.

### Support modal (новый)
- Список тикетов (`/api/support/list`).
- Кнопка «Новое обращение» — форма (тема + категория из 8 значений + текст).
- Открытие тикета — стилизованный чат-вид (`/api/support/messages`), пузырьки админа/юзера.
- Отправка ответа (`/api/support/reply`).

### AI Tools modal (новый)
- Таб «AI Чат» — пузырьковый чат через `/api/tools/ai_chat`, история в памяти сессии.
- Таб «Генератор описаний» — проверка cookie через `/api/roblox/check_cookie`, после успеха форма параметров (стиль/доп. пожелания) и сабмит на `/api/tools/generate_description` с кнопкой копирования результата.

### Robux calculator (расширен)
В существующий `ROBUX_CALC_HTML()` добавлены:
- Табы режима «По нику» / «По ссылке».
- Кнопка «Проверить» рядом с «Купить» — вызов `/api/robux/inspect` с показом мини-карточки геймпасса (картинка / имя / цена).
- Кнопка «Купить» **disabled** до успешной проверки. Это устраняет ошибки `gamepass not found` после оформления.
- В режиме URL — отдельное поле для ссылки и Robux.

### Mobile bottom nav (новая)
Скрыта на десктопе (>760px). На мобилке — фиксированная панель: Главная / Кейсы / AI / Помощь / Профиль.
