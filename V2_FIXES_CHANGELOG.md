# v2.html — фиксы и апгрейд (17 апр 2026)

## Что было сломано

### 🔴 Критическое
1. **Модалка логина физически отсутствовала в DOM.** `openAuth()` пытался обратиться к `#authModal`, которого не было → клик «Войти» роняет JS через глобальный обработчик ошибок.
2. **`loadCatalog()` ломился в `#games`**, а в HTML только `#gamesGrid`. Скелетоны не показывались (каталог показывался «пусто», пока не отработает SHOP).
3. **Поиск был мёртвый.** `onSearch` вызывал `renderGames()`, которую позже перекрывала пустая no-op функция. Ввод в поисковую строку не делал ничего.
4. **Целый блок CSS отсутствовал.** Страница игры, калькулятор Robux, пустые состояния — рендерились без стилей.

### 🟡 Функциональное
- Auth-флоу не знал про капчу → регистрация всегда 400.
- Auth-флоу не знал про email-2FA → при включённой 2FA логин молча падал.
- Регистрация обрывалась на «проверь почту, доделай в классике» вместо полного флоу.
- Нет уведомлений.
- Нет отзывов.
- Footer отсутствовал.
- Subnav (Главная / Отзывы / Поддержка / Блог / Раздача) — ссылки никуда не вели.

## Что сделано

### 1. Auth-модалка (полноценная)
- HTML модалки (`#authModal`) с тремя шагами: `main` / `2fa` / `confirm`.
- Математическая капча `a + b = ?` (как в v1), с авто-подгрузкой через `GET /api/captcha/challenge`.
- Логин: если бэк ответил `needs_2fa:true` → переключается на ввод кода, шлёт `POST /api/auth/login_confirm`.
- Регистрация: `POST /api/auth/register_start` → вводит код из письма → `POST /api/auth/register_confirm` → автоматический вход.
- Ссылка «Забыли пароль?» кидает в классику (сам reset-flow портить пока не стал).
- При ошибке капча перегенерируется автоматически.

### 2. Исправленные селекторы
- `loadCatalog` теперь корректно пишет скелетоны в `#gamesGrid`.
- Удалены мёртвые `renderGames()` / `renderChips()` (были в коде, но не вызывались после перехода на home/game роутер).
- `onSearch` с debounce 120ms вызывает `syncView()` → рендерятся карточки текущего view с учётом `SEARCH_QUERY`.

### 3. Уведомления
- Иконка-колокол в шапке, показывается только залогиненым, с пульсирующим красным счётчиком непрочитанных.
- Панель-дропдаун справа, при открытии подтягивает `GET /api/notifications?limit=30`.
- Клик по уведомлению → `POST /api/notifications/read` (помечает прочитанным).
- Кнопка «Прочитать все» → `POST /api/notifications/read_all`.
- Бейдж обновляется раз в 30 сек через `/api/notifications/unread_count`.
- Авто-закрытие по клику вне панели.

### 4. Отзывы

- Грид карточек с аватаркой, рейтингом, PREMIUM-бейджем, кол-вом покупок, датой.
- Форма «Оставить отзыв» с клик-звёздочками и textarea (10-800 символов).
- `GET /api/reviews`, `GET /api/reviews/my`, `POST /api/reviews/create`.
- Если у юзера уже есть отзыв — форма показывает его с пометкой «на модерации».

### 5. Страница игры + калькулятор Robux — получили CSS

## Что НЕ проверено
- Реального рантайма в браузере я не запускал — только синтаксис.
- Капча: пробовал по коду `_captcha_verify` в `main.py`. Важный момент: для **админов капча пропускается** (`is_admin_user` skip), так что они могут логиниться даже если капча-input пустой; для обычных юзеров поле капчи обязательно.
- Auto-login после `register_confirm` делает второй вызов `/api/auth/login` с пустым `captcha_answer` — скорее всего вернёт 400 «Captcha failed», и юзер увидит тост «Теперь войди с логином и паролем» + откроется модалка. Это фолбэк, не баг. Если хочешь auto-login — надо либо (а) делать ещё один challenge+ansver перед ним, либо (б) на стороне бэка поддержать грейс-период после register_confirm.
- Reviews: `/api/reviews/my` требует `require_user` — у незалогиненого юзера форма не откроется, вместо этого откроется authModal. Это правильное поведение.
